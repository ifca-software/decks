Best Practices

Tips from Boris Cherny, creator of Claude Code

Direct quotes from the person who built the tool.

CLAUDE.md

"Anytime we see Claude do something incorrectly we add it to the CLAUDE.md, so Claude knows not to do it next time."

Source →

Plan Mode

"Most sessions start in Plan mode. I go back and forth with Claude until I like its plan, then switch into auto-accept — Claude can usually one-shot it."

Source →

Verify

"Give Claude a way to verify its work. If Claude has that feedback loop, it will 2-3x the quality of the final result."

Source →

Use Opus

"It's the best coding model I've ever used... you have to steer it less and it's better at tool use, it is almost always faster than using a smaller model."
